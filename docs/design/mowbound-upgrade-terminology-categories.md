# Mowbound 업그레이드 용어와 3갈래 카테고리 정리

작성일: 2026-06-22 KST

## 결론

Mowbound의 성장 시스템 명칭은 `스킬`보다 `업그레이드`가 맞다.

- `스킬`은 RPG, 전투, 캐릭터 숙련도, 액티브 능력 느낌이 강하다.
- `업그레이드`는 인크리멘탈 게임의 반복 구매, 성능 증가, 장비 해금, 경제 효율 증가와 잘 맞는다.
- Mowbound는 골드로 장비, 수확 효율, 환경 콘텐츠를 해금하는 구조이므로 `업그레이드 트리`가 더 정확하다.

UI 권장 표기:

| 위치 | 권장 표기 |
|---|---|
| 메인 메뉴 버튼 | 업그레이드 |
| 화면 제목 | 업그레이드 트리 |
| 구매 버튼 | 해금 또는 업그레이드 |
| 잠긴 카드 | 필요 골드 / 조건 미달 |
| 외계 문양, 레이저, 트랙터 | 특수 업그레이드 또는 고유 이름 |

## 3갈래 기획 카테고리

카테고리명 `장비`, `수확`, `환경`은 기획/배치용 분류명이다. 실제 업그레이드 이름에 접두어로 붙이지 않는다.

| 카테고리 | 플레이어가 느끼는 역할 | 포함 방향 |
|---|---|---|
| 장비 | 더 잘 벤다 | 낫, 큰낫, 공격력, 공격범위, 공격속도, 예초기, 트랙터, 레이저 |
| 수확 | 더 많이 번다 | 골드 증가, 점수 증가, 깔끔한 잔디 보너스, 클리어 보너스, 게임 중 이속 증가, 플레이타임 증가 |
| 환경 | 더 넓고 복잡해진다 | 신규 풀, 질긴 풀, 돌 파괴, 나무 베기, 폭탄, 장애물 보상, 맵 오픈 |

## 첫 해금 이후 구조

첫 업그레이드 이후에는 너무 많은 노드가 한 번에 열리지 않게 한다. 시작은 3갈래만 보인다.

```txt
첫 업그레이드
├─ 장비 계열 첫 노드
├─ 수확 계열 첫 노드
└─ 환경 계열 첫 노드
```

예상 배치:

| 갈래 | 첫 노드 후보 | 이후 파생 |
|---|---|---|
| 장비 | 날 세우기 또는 낫 개량 | 큰낫, 빠른 회전날, 예초기, 트랙터, 레이저 |
| 수확 | 시장 수레 또는 수확 장부 | 깔끔한 잔디, 클리어 보너스, 황금 들판, 긴 하루, 수확 질주 |
| 환경 | 밭 개척 또는 질긴 풀 | 돌 파괴, 나무 베기, 폭탄 배치, 넓은 밭 계약 |

## 명칭 원칙

- 시스템명은 `업그레이드`.
- 그래프 UI는 `업그레이드 트리`.
- 개별 노드는 고유한 세계관 이름을 쓴다.
- `장비/수확/환경`은 트리 배치, 색, 아이콘, 내부 branch id에만 사용한다.
- 카드 제목에 `장비 업그레이드: 날 세우기`처럼 쓰지 않는다.
- 필요하면 트리 영역 가장자리나 필터 버튼에만 `장비`, `수확`, `환경` 라벨을 쓴다.

## 구현 영향

현재 코드에는 `SkillTree`, `SkillNode`, `skillName` 같은 `skill` 용어가 남아 있다. UI 문구는 먼저 `업그레이드`로 바꾸고, 내부 타입/파일명은 별도 리팩터링으로 분리하는 것이 안전하다.

권장 순서:

1. UI 문구: `스킬 트리` -> `업그레이드 트리`, `스킬` -> `업그레이드`.
2. 스킬 데이터 branch를 `equipment`, `harvest`, `environment` 중심으로 재배치.
3. 첫 해금 뒤 공개 노드를 3개로 제한.
4. progression 테스트를 새 해금 순서에 맞게 갱신.
5. 내부 코드명 `Skill*` -> `Upgrade*`는 별도 PR에서 처리.

## 결정

Mowbound는 앞으로 플레이어-facing 용어로 `업그레이드`를 사용한다. 초기 트리 구조는 `장비 / 수확 / 환경` 3갈래로 재구성한다.
